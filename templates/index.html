How It Works

Bloom Filter
Probabilistic bit-vector fingerprint
▼

Each password is compressed into a 1 000-bit fingerprint using bi-grams and hash functions — the raw password is never stored.

Step 1 · Pad → extract bi-grams
Step 2 · f = SHA-256(b),  g = MD5(b)
Step 3 · hi(b) = (f + i·g) mod 1000
Step 4 · Set bits h0 … h19 = 1

Parameters: L = 1 000 bits  ·  k = 20 hash functions

Jaccard Coefficient
Primary decision metric — threshold ≥ {{ threshold }}
▼
⚡ Primary — Jaccard ≥ {{ threshold }} → REJECT

Measures the overlap as a fraction of the union of all set bits in the two filters.

J(β1, β2) =   γ   /  (kβ1 + kβ2 − γ)
γpositions where both filters have a 1 (bitwise AND count)
kβtotal number of 1-bits in a filter
Worked Example
kβ₁ = 160,  kβ₂ = 175,  γ = 48
J = 48 / (160 + 175 − 48) = 48 / 287 = 0.167 → ACCEPT
🎲
Dice Coefficient
Reference metric
▼
Reference only

Like Jaccard but doubles the shared bits, giving extra credit to intersection over the total size of both filters.

δ(β1, β2) = 2γ / (kβ1 + kβ2)
2γshared bits, counted twice — rewards dense overlap
kβtotal 1-bits in each filter
Same Values as Above
δ = (2 × 48) / (160 + 175) = 96 / 335 = 0.286
Dice ≥ Jaccard always — it penalises the union less.
📐
Cosine Similarity
Reference metric
▼
Reference only

Treats each filter as a binary vector. Computes the cosine of the angle between the two vectors in 1 000-dimensional space.

φ(β1, β2) = γ / (√kβ1 · √kβ2)
√ksquare root of 1-bit count = vector magnitude
γdot product of the two bit vectors
Same Values as Above
φ = 48 / (√160 × √175) = 48 / (12.65 × 13.23) = 48 / 167.4 = 0.287
Cosine ≈ Dice for balanced vectors.

Critical Analysis
False positives, bit saturation & trade-offs
▼
Learning Outcome 5 — Discussing Results Critically

Bloom filters are probabilistic: they trade a small probability of error for extreme speed and low memory. Two types of error can occur in this system.

False +
False Positive — an unrelated password is wrongly rejected. Different bi-grams can hash to the same bit positions via hi = (f + i·g) mod 1000, causing an unrelated filter to share enough bits by coincidence to cross the threshold.
False −
False Negative — a tweaked password is wrongly accepted. This happens when a mutation changes enough bi-grams that the Jaccard score drops below 0.30. No false negatives were observed on the 10-pair test set (minimum similar score: 0.59).
Bit Saturation
Each password sets up to k × bigrams = 20 × ~10 = 200 bit positions in the L = 1 000 vector. As the dataset grows, more bits become 1, and any two unrelated passwords start sharing more positions by chance — raising the baseline Jaccard score and increasing false positive rate. The current 250-password dataset keeps saturation moderate; scaling to millions of passwords would require a larger L.
Threshold Trade-off
Lower (e.g. 0.20) Fewer false negatives — harder to sneak in tweaked passwords. But more false positives — legitimate new passwords rejected more often.
Higher (e.g. 0.50) Fewer false positives — better user experience. But more false negatives — close tweaks like passw0rd may slip through.
At 0.30: margin of 0.12 above the different-cluster ceiling (0.18) and 0.29 below the similar-cluster floor (0.59) — intentionally asymmetric to stay safe as bit saturation grows.
